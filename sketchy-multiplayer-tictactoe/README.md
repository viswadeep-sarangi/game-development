# TIC TAC TOE - Online Multiplayer
## This game consists of 2 parts, (1) Server, (2) Client
----------------------------
### 📡 (1) FastAPI WebSocket Tic Tac Toe Server
This is a lightweight, server-authoritative backend for a multiplayer Tic Tac Toe game, built using FastAPI and WebSockets. The server supports room-based gameplay and real-time communication between two players.

🔗 Clients can be built in any engine or framework that supports WebSocket (e.g., Godot, Unity, JavaScript).

🚀 Features
✅ Real-time 2-player WebSocket communication

✅ Room-based multiplayer via {game_id}

✅ Server-authoritative move validation

✅ Win/draw detection and broadcast

✅ Lightweight and production-ready

✅ Easily deployable on Fly.io
```
📁 Project Structure
server/
├── main_server.py             # FastAPI WebSocket entrypoint
├── game_server_manager.py     # Game state and logic manager
├── requirements.txt    # Python dependencies
├── Dockerfile          # Docker config for Fly.io deployment
├── Makefile            # Local automation (optional)
└── fly.toml            # Auto-generated by `fly launch`
```

🧪 Local Development
1. Create a virtual environment and install dependencies
```
python -m venv .venv
source .venv/Scripts/activate   # or `.venv/bin/activate` on Linux/macOS
```
```
pip install -r requirements.txt
```
2. Run the server locally
```
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
The server will be available at:

```
ws://localhost:8000/ws/{game_id}/{player_id}
```
You can test it using:

**Piesocket WebSocket Tester**

Or any WebSocket client (including your game)
```
🕹️ Message Protocol
Message Sent	Meaning
`MOVE	row
`UPDATE	row
`WIN	player_id`
DRAW	Game ended in a draw
`JOIN	player_id`
```
☁️ Deploying on Fly.io
1. Install the Fly.io CLI
```
curl -L https://fly.io/install.sh | sh
```
2. Log in and initialize
```
fly auth login
fly launch
```
Choose a name and region, and decline the Postgres option.

3. Deploy your server
```
fly deploy
```
After deployment, your WebSocket server will be live at:

```
wss://<your-app-name>.fly.dev/ws/{game_id}/{player_id}
```
📦 Docker (Optional Manual Deployment)
To build and run the server manually via Docker:

```
docker build -t tictactoe-server .
docker run -p 8000:8000 tictactoe-server
```
